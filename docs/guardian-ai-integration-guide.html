Protocol Integration Guide — how to connect any Sui DeFi protocol to GuardianAI's autonomous on-chain risk engine.

Overview

GuardianAI watches your protocol's price feeds, liquidity levels, and on-chain state in real time. When the risk score crosses a threshold, the agent executes a pre-authorised defensive action directly on-chain — in seconds, not minutes.

The permission model is enforced by Move's type system, not application logic. The agent can only call the exact functions you expose to it. Nothing else.

Why Sui specifically? On Ethereum, giving an AI agent on-chain permissions requires complex proxy contracts, multisigs, and governance delays. On Sui, Move's capability objects enforce permissions at the VM level. It's a hard guarantee, not a promise.

1

Add GuardianAI as a Move Dependency

In your protocol's Move.toml:

[dependencies]
GuardianAI = {
  git = "https://github.com/your-org/guardian-ai",
  subdir = "contracts/guardian_ai"
}
2

Add Two Functions to Your Contract

This is the only code your team writes. Add these to your existing protocol contract: 

use guardian_ai::cap::{GuardianCap, GuardianConfig};
use guardian_ai::cap;

// 1. Accept and store the capability
public fun enable_guardian(cap: GuardianCap, ctx: &mut TxContext) {
    // store cap in your protocol's shared object
}

// 2. Protected circuit breaker — GuardianCap is the key
//    Move enforces that ONLY the GuardianCap holder can call this
public fun pause_swaps(
    _cap: &GuardianCap,
    config: &GuardianConfig,
    protocol: &mut Protocol,
) {
    cap::assert_active(_cap, config); // checks guardian is enabled
    protocol.swaps_enabled = false;
}

The _cap: &GuardianCap parameter is the critical part. Move's type system enforces at the VM level that only the holder of a GuardianCap object can invoke this function. If you don't have it, the transaction is rejected before it runs.

3

Sign One Transaction to Activate

The protocol team calls guardian_ai::cap::initialize(agent_wallet_address) once. This automatically creates three objects:

Object Goes To Purpose
GuardianCap Agent wallet Grants narrow permission to call circuit breaker functions
AdminCap Protocol team wallet Full control — enable, disable, override the guardian at any time
GuardianConfig Shared (public) Live guardian state — anyone can verify it on-chain

Agent Permissions

The agent's powers are narrow by design. Move's type system makes this a hard enforcement, not a configuration setting.

Action Agent Can Do It
Call circuit breaker functions (pause, tighten LTV) ✓ Yes
Withdraw funds ✗ No
Change protocol ownership ✗ No
Call any function not explicitly exposed with &GuardianCap ✗ No
Act after AdminCap revokes the guardian ✗ No

Risk Scoring

The off-chain engine scores risk every 4 seconds from Pyth price feeds and on-chain state. Score is capped at 100.

Signal How Detected Score Added
Price deviation >10% from TWAP Pyth Hermes API comparison +30
Price deviation 3–10% from TWAP Pyth Hermes API comparison +15
Oracle stale >30s Pyth publish timestamp check +20
Pool balance dropped >20% this interval On-chain object polling delta +35
Pool balance dropped 5–20% this interval On-chain object polling delta +15
Pool critically low (<1% of baseline) Absolute balance check +90
Protocol already paused on-chain Protocol object state read +10
Score Action Taken
0 – 49 Log only — no notification
50 – 69 Notify protocol team (Telegram / webhook)
70 – 84 Tighten LTV ratio via on-chain transaction
85 – 100 Pause protocol via on-chain transaction

Human Override

The protocol team is always in control. The AdminCap gives instant override capability: 

// Disable the guardian instantly — one transaction
guardian_ai::cap::disable(&admin_cap, &mut guardian_config);

// Unpause the protocol after review
your_protocol::unpause_protocol(&admin_cap, &mut protocol, &mut log, &clock);

// Re-enable when ready
guardian_ai::cap::enable(&admin_cap, &mut guardian_config);

Every action is permanently logged on-chain. The ActionLog shared object records every guardian action with timestamp, risk score, and plain-English reasoning. Full signal context is stored on Walrus. No black box.
